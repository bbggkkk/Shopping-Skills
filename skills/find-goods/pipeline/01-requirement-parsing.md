# Step 1: 요구사항 분석

## 개요

사용자로부터 상품 요구사항을 분석하고, 검색에 필요한 키워드와 검증 기준을 추출합니다.

## 입력

사용자로부터 다음 정보를 수집합니다:

| 항목 | 필수 | 설명 |
|------|------|------|
| **상품명** | O | 찾고자 하는 상품의 이름 |
| **브랜드** | X | 선호하는 브랜드 |
| **모델번호** | X | 특정 모델 번호 |
| **카테고리** | X | 상품 분류 (전자제품, 의류 등) |
| **사양/스펙** | X | 요구하는 기능이나 사양 |
| **예산** | X | 희망 가격 범위 |
| **기타 요구사항** | X | 추가 조건 (색상, 크기 등) |

## 처리 과정

### 1단계: 입력 파싱

사용자 입력에서 핵심 정보를 추출합니다:

```
입력: "삼성 갤럭시북 프로 360 15인치 i7 16GB 찾기"

추출 결과:
- 상품명: 갤럭시북 프로 360
- 브랜드: 삼성
- 사양: 15인치, i7, 16GB
- 카테고리: 노트북
```

### 2단계: 검색 키워드 생성

추출된 정보를 기반으로 검색 키워드를 생성합니다:

| 키워드 유형 | 형식 | 예시 |
|-------------|------|------|
| **기본 검색** | [상품명] [브랜드] | "갤럭시북 프로 360 삼성" |
| **상세 검색** | [상품명] [모델번호] [사양] | "갤럭시북 프로 360 NT950QDX-G516 15인치 i7 16GB" |
| **대안 검색** | [상품명] [유사제품] | "갤럭시북 프로 360 vs 엘리엇북 프로" |

### 3단계: 카테고리 분류

상품 카테고리를 자동 분류합니다:

| 카테고리 | 키워드 예시 |
|----------|-------------|
| **노트북** | 노트북, 북, 랩탑, 울트라북 |
| **스마트폰** | 폰, 스마트폰, 휴대폰 |
| **태블릿** | 태블릿, 아이패드, 갤럭시탭 |
| **가전** | 냉장고, 세탁기, 에어컨, TV |
| **의류** | 옷, 티셔츠, 바지, 자켓 |
| **식품** | 음식, 식료품, 과자, 음료 |
| **도서** | 책, 서적, 만화 |

### 4단계: 검증 기준 설정

카테고리에 따라 검증 기준을 자동 설정합니다:

```yaml
validation_criteria:
  electronics:
    stock_check: true
    spec_match: true
    price_consistency: true
    warranty_check: true
  
  clothing:
    stock_check: true
    size_match: true
    color_match: true
    price_consistency: true
  
  food:
    stock_check: true
    expiry_check: true
    price_consistency: true
```

## 규칙

1. **최소 1개 필수 항목** — 상품명은 반드시 있어야 함
2. **키워드 중복 제거** — 동일한 키워드가 여러 번 생성되지 않도록
3. **카테고리 자동 분류** — 명시적이지 않은 경우 키워드로 추론
4. **검증 기준 기본값** — 카테고리 미지정 시 일반적인 기준 적용
